# 数据口径查询助手

从指标字典 Excel 中提取指标信息，通过 TF-IDF + 余弦相似度实现中文模糊搜索，快速定位指标的业务口径、技术口径和 SQL 逻辑。

## 解决什么问题

指标字典散落在多个 Excel 文件中，查找某个指标的口径定义需要反复翻表。本工具将多份 Excel 合并加载，提供统一的搜索入口，输入关键词即可匹配相关指标。

## 数据源配置

所有数据源在 `sources.yaml` 中配置。新增 Excel **只需加一条配置，不用改代码**。

Excel 文件放在 `data/` 目录下，已通过 `.gitignore` 排除，不会提交到仓库。

### 什么是 YAML

`sources.yaml` 是一个 YAML 格式的配置文件。YAML 是一种用缩进表示层级的文本格式，比 JSON 更适合人手动编辑——不需要写大括号、方括号和逗号。

编辑注意事项：
- **用空格缩进，不能用 Tab**（建议 2 个空格一级）
- 冒号后面要有一个空格：`key: value` ✅　`key:value` ❌
- 以 `- ` 开头表示列表中的一项
- `#` 后面是注释

### 新增数据源

1. 把 Excel 文件放进 `data/` 目录
2. 打开 `sources.yaml`，在 `sources:` 下面追加一条配置：

```yaml
  - file: 新项目指标.xlsx       # Excel 文件名，放在 data/ 下
    sheet: Sheet1               # Sheet 名称
    biz_line: 新业务线           # 业务线标签
    source_tag: 新项目指标.xlsx  # 搜索结果中显示的来源
    columns:                    # 列名映射：左边是系统字段，右边是 Excel 中的实际列名
      指标名称: 指标名           # （必填）其余选填
      指标编码: 编码
      类型: _literal:指标        # _literal: 前缀表示固定值，不从 Excel 读取
      业务口径: 口径说明
      技术口径: 技术说明
      SQL: sql逻辑
      应用报表: 报表名
```

3. 重启应用即可生效

### 配置字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `file` | ✅ | Excel 文件名，放在 `data/` 目录下 |
| `sheet` | ✅ | 要读取的 Sheet 名称 |
| `biz_line` | ✅ | 业务线标签，用于搜索时筛选 |
| `source_tag` | 否 | 搜索结果中显示的来源标识，默认用文件名 |
| `skip_rows` | 否 | 跳过前 N 行（有些表第一行是合并单元格标题） |
| `skip_values` | 否 | 指标名称列中需要跳过的值（如分类标题行） |
| `columns` | ✅ | 列名映射，见下方说明 |

**columns 映射规则**：

左边是系统字段名，右边是 Excel 中的实际列名。系统字段包括：

| 系统字段 | 必填 | 说明 |
|----------|------|------|
| `指标名称` | ✅ | 对应 Excel 中指标名称的列 |
| `指标编码` | 否 | 指标编码 |
| `类型` | 否 | 指标类型 |
| `业务口径` | 否 | 业务口径定义 |
| `技术口径` | 否 | 技术口径定义 |
| `SQL` | 否 | SQL 逻辑。支持 `SQL_fallback` 配置备选列名 |
| `应用报表` | 否 | 应用报表 |

特殊写法：
- `类型: _literal:指标` — 以 `_literal:` 开头表示所有行都填固定值"指标"，不从 Excel 读取
- `SQL_fallback: 开发逻辑` — 当 `SQL` 对应的列为空时，回退读取这个列

### 列名含换行符/特殊空白的处理

有些 Excel 的列名里会包含换行符，比如实际列名是：

```
开发逻辑
（填线上版本的逻辑）
```

在 YAML 里直接写成一行即可：

```yaml
SQL: 开发逻辑\n（填线上版本的逻辑）
```

程序会自动做**模糊列名匹配**——去掉所有空白字符和 `\n` 后再比较，所以不需要在 YAML 里精确还原换行符。写成 `开发逻辑（填线上版本的逻辑）` 也能匹配上。

## 搜索原理

**整体流程**：加载所有 Excel → 把每条指标的文本变成数字向量 → 搜索时比较"方向是否接近"

**分步说明**：

1. **数据加载**：读取所有 Excel，把每条指标的「指标名称 + 业务口径 + 技术口径」拼成一段文本
2. **中文分词（jieba）**：把中文文本切成词语。比如"日均有效播放量"会被切成"日均 / 有效 / 播放量"，这样才能逐词分析
3. **TF-IDF 向量化**：给每个词算一个权重分数，变成一组数字（向量）
   - TF（词频）：这个词在当前指标里出现越多，分越高
   - IDF（逆文档频率）：这个词在所有指标里都出现（比如"指标""数据"），说明它没有区分度，分就低
   - 效果：真正能代表这条指标特征的词权重高，常见废话权重低
4. **余弦相似度**：搜索时，把你输入的关键词也用同样方式变成向量，然后和每条指标的向量比较"方向"是否接近
   - 两个向量方向越一致，相似度越接近 1，说明越相关
   - 完全无关则接近 0
   - 不看向量长度，只看方向——所以短查询也能匹配长文本
5. **排序返回**：按相似度从高到低排，取前 10 条

## 快速开始

```bash
# 安装依赖
pip install -r requirements.txt

# 启动 Web 界面（Gradio）
python app.py
# 浏览器打开 http://localhost:7860

# 或使用命令行交互模式
python metric_search.py
```

### 多 Python 版本注意

如果系统装了多个 Python 版本，需要用安装了依赖的那个版本来运行。比如依赖装在 Python 3.11 下，就要用对应路径启动：

```powershell
# PowerShell 示例
cd "D:\油茶茶与虾滑滑\SYSU\研一下\万声\探索\数据口径查询"
D:\python\Python311\python.exe app.py
```

确认方法：用目标 Python 执行 `python -c "import pandas, jieba, sklearn, gradio, yaml"`，没报错就说明依赖齐全。

## 搜索功能

- **关键词搜索**：输入指标名称或相关描述，如「有效播放」「日活」「留存率」
- **精确匹配优先**：指标名称包含搜索词的结果自动置顶
- **业务线筛选**：按业务线过滤（业务线来自 sources.yaml 配置）
- **指标类型筛选**：按指标、概念定义等类型过滤
- **结果展示**：指标名称、编码、业务口径、技术口径、SQL（支持高亮）、应用报表、数据来源
- **加载异常提示**：文件缺失或 Sheet 读取失败时，页面和命令行都会显示具体警告

## 技术栈

- **pyyaml** — 读取 `sources.yaml` 配置文件的 Python 库（YAML 是一种易读的配置文件格式）
- **pandas + openpyxl** — Excel 读取与数据处理
- **jieba** — 中文分词
- **scikit-learn** — TF-IDF 向量化 + 余弦相似度计算
- **Gradio** — Web 界面

纯本地运行，不依赖外部 API。
